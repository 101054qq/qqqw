## 🔑 核心設計原則

### 1. 合約優先 (Contract-First)

所有型別從 Zod Schema 推導，永不過期。`shared/contracts.ts` 是前後端唯一的真實來源。

```typescript
// ✅ 型別自動推導，無需手動維護兩份
export const menuItemSchema = z.object({
  id: z.number().int().min(1),
  name: z.string().min(1),
  price: z.number().min(0),
  category: z.string().min(1),
  description: z.string(),
  image_url: z.string().min(1),
});
export type MenuItem = z.infer<typeof menuItemSchema>;
```

### 2. 錯誤處理 — Result<T, E> 模式

```typescript
// ❌ 舊：10+ 重複 if 語句
if (!result.ok && result.code === "ORDER_NOT_FOUND") { set.status = 404; ... }
if (!result.ok && result.code === "MENU_ITEM_NOT_FOUND") { set.status = 404; ... }
if (!result.ok && result.code === "ORDER_NOT_OWNED") { set.status = 403; ... }
if (!result.ok && result.code === "ORDER_NOT_EDITABLE") { set.status = 409; ... }
if (!result.ok && result.code === "EMPTY_ORDER") { set.status = 400; ... }

// ✅ 新：錯誤碼映射表
const ERROR_MAP: Record<string, number> = {
  ORDER_NOT_FOUND: 404,
  MENU_ITEM_NOT_FOUND: 404,
  ORDER_NOT_OWNED: 403,
  ORDER_NOT_EDITABLE: 409,
  EMPTY_ORDER: 400,
} as const;

function handleStoreResult(result, set) {
  if (!result.ok) {
    set.status = ERROR_MAP[result.code] ?? 500;
    return { error: ERROR_MESSAGES[result.code] ?? "Unexpected store state" };
  }
  return { data: toOrderResponse(result.order) };
}
```

### 3. 路由分離

每個路由檔案只負責一個資源（menu / orders / auth），共用中介層提取到 `middleware/`。

### 4. 前端狀態管理

- Context API 替代 12+ 個獨立 useState
- ErrorBoundary 優雅處理 API 錯誤
- Skeleton UI 改善加載體驗

## 🗄️ 資料庫 Schema (bf_v9)

```sql
-- 菜單
CREATE TABLE bf_v9.menu_items (
  id          SERIAL PRIMARY KEY,
  name        TEXT NOT NULL,
  price       INTEGER NOT NULL,
  category    TEXT NOT NULL,
  description TEXT NOT NULL,
  image_url   TEXT NOT NULL
);

-- 訂單
CREATE TABLE bf_v9.orders (
  id           SERIAL PRIMARY KEY,
  user_id      TEXT NOT NULL REFERENCES bf_v9.user(id),
  total        INTEGER NOT NULL DEFAULT 0,
  status       TEXT NOT NULL DEFAULT 'pending',
  created_at   TIMESTAMPTZ NOT NULL,
  submitted_at TIMESTAMPTZ
);
-- 🆕 複合索引
CREATE INDEX orders_user_status_idx ON bf_v9.orders(user_id, status);

-- 訂單項目（反正規化，保留點餐時的菜單快照）
CREATE TABLE bf_v9.order_items (
  id          SERIAL PRIMARY KEY,
  order_id    INTEGER NOT NULL REFERENCES bf_v9.orders(id) ON DELETE CASCADE,
  item_id     INTEGER NOT NULL,
  name        TEXT NOT NULL,
  price       INTEGER NOT NULL,
  category    TEXT NOT NULL,
  description TEXT NOT NULL,
  image_url   TEXT NOT NULL,
  qty         INTEGER NOT NULL,
  UNIQUE (order_id, item_id)
);
```
